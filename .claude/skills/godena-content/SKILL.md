# Godena content drafting

Goal: 3-5 ready-to-post drafts per week that earn attention honestly. Run `godena-monitor` FIRST so posts use real numbers (build-in-public only works when true).

## Weekly set (draft these)
1. **Build-in-public update** (X + LinkedIn): one real thing that changed this week + one real number (agents indexed, a new category, a directory listing). Ex: *"Godena now indexes 8,300+ AI agents & real services — search 'video ai' or 'lawyer kampala', get the best match ranked by trust. Open source. sammyghe.github.io/Godena"*
2. **Feature demo** (X, short): a single search result screenshot/description — *"Asked Godena for 'research ai' → Deep Research agents, ranked. Try it."*
3. **Dev-audience post** (dev.to or X thread): a technical angle — "an unkillable git-native search engine", "how Godena is discoverable BY other agents (MCP + A2A card)", "$0 infra".
4. **Africa/services angle** (for WhatsApp groups / LinkedIn, once messaging is live): *"Find a trusted lawyer/pharmacy/mechanic in Kampala by texting one bot."* — human, relationship-led.

## Rules
- **Every claim true.** No "10k users" — say what's real (8,300 agents indexed; pre-users).
- Lead with the live link. One clear CTA.
- Sentence case, no hype words, no emoji spam. Value/utility first.
- Reuse the canonical definition + drafts in `Godena-strategy/POSTS.md`.

## Output
Write the drafts into `Godena-strategy/POSTS.md` under a dated "This week" heading, labeled by platform. Flag which need an image. Samuel reviews + posts.
