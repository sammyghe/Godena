# Godena weekly growth cycle

Run these in order, produce ONE report. Each is a self-contained skill in `.claude/skills/`.

1. **`godena-monitor`** — pull the exposure dashboard (stars, traffic, web visitors, listings, mentions). This is the truth check.
2. **`godena-seo`** — verify/refresh discoverability (topics, meta, sitemap, keywords). Usually already good — only fix regressions.
3. **`godena-submit`** — advance the directory/registry submission tracker; produce the next batch to submit (Samuel submits, or drafts the submission).
4. **`godena-content`** — draft this week's posts (build-in-public update using the real monitor numbers, one feature demo, optionally a dev.to article). Output = ready-to-post drafts.
5. **`godena-outreach`** — draft 3-5 value-first outreach messages (agent builders, directory owners, one community). Samuel sends.

## Rules
- **Monitor first, then draft using the real numbers** — build-in-public works because the numbers are real.
- **Draft only; never post or create accounts** — that's Samuel's click (compliance + anti-ban).
- **No fake traction, no spam.** Value-first everywhere.
- Two tracks, labeled: **dev/AI-ecosystem** (GitHub/HN/MCP/dev.to — automatable) and **Africa/services** (WhatsApp/local — human, needs the Telegram channel live).

## Output
Append the exposure line to `Desktop\Godena-strategy\STATE.md`, then list: this week's drafted posts, submissions to make, outreach to send, and the 1 thing Samuel should do. Escalate to Samuel only what needs an account/publish/spend.
